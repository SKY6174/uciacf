# 울산과학대학교 산학협력단 성과관리 대시보드 계획서

> **요약**: 산학협력단장이 조직·사업·재정·성과·문서를 한곳에서 조회하고 중장기 발전방안을 수립할 수 있는 통합 IR 대시보드 구축 계획
>
> **프로젝트**: 울산과학대학교 산학협력단 통합 성과관리

## 1. 배경과 목적

울산과학대학교 산학협력단은 산학기획팀, 산학지원팀, 부속기관, 기업인재교육본부, 사업기구, 학교기업, 국책사업단 등 다수 조직과 사업을 운영한다. 사업별 시스템과 문서 폴더에 흩어진 정보를 단장이 필요할 때 즉시 찾고, 예산·성과·일정·위험을 같은 기준으로 비교할 수 있는 경영 의사결정 환경이 필요하다.

본 프로젝트는 기존 앵커사업 통합 IR 대시보드의 익숙한 카드·차트·연도 선택 경험을 확장해 산학협력단 전체를 포괄하는 단일 관제 화면을 구축하는 것이 목적이다.

## 2. 사용자와 핵심 질문

| 사용자 | 필요한 판단 |
|---|---|
| 산학협력단장/부단장 | 전체 조직의 예산, 핵심성과, 위험, 일정은 어떠한가? 어디에 개입해야 하는가? |
| 본부장/팀장/센터장/사업단장 | 담당 조직의 목표 대비 실적과 마감 업무는 무엇인가? |
| 실무 담당자 | 어떤 자료를 언제, 어떤 기준으로 입력·갱신해야 하는가? |
| 감사/재정/성과 담당자 | 수치의 근거, 변경 이력, 증빙 문서는 무엇인가? |

## 3. 목표

- [ ] 조직도 기반으로 모든 조직·사업단을 탐색하고 현황을 드릴다운한다.
- [ ] 회계연도 및 사업연도별 예산·집행·성과지표를 통합 비교한다.
- [ ] 협약, 연구지원, 지식재산권, 가족회사, 일학습병행, 학교기업, 국책사업 등 핵심 업무를 표준 구조로 관리한다.
- [ ] 회의, 규정, 보고서, 증빙 문서를 해당 조직·사업·지표에 연결한다.
- [ ] 단장용 요약, 조직책임자용 운영, 실무자용 입력 화면을 권한별로 제공한다.
- [ ] 추세와 위험 신호를 근거 데이터까지 추적 가능한 형태로 제시한다.

## 4. 범위

### 4.1 1차 구축 범위(MVP)

- 인증 및 역할 기반 접근 제어
- 조직·조직도·사업단 마스터 관리
- 통합 IR 홈: 예산, 집행률, KPI 달성률, 마감 일정, 경보
- 조직/사업 상세: 기본정보, 책임자, 예산, KPI, 일정, 문서, 최근 활동
- 업무 탐색 트리: 조직 → 업무영역 → 기록/회의/사업/문서
- 예산·집행 관리 및 월별 스냅샷
- KPI 정의·목표·실적·근거자료 관리
- 회의/위원회, 일정/마감, 문서 메타데이터 관리
- CSV/XLSX 초기 데이터 반입과 엑스포트
- 변경 이력 및 데이터 최신성 표시

### 4.2 2차 확장 범위

- 연구과제·산학협력 수익·기술이전·특허·가족회사·현장실습 등 도메인별 상세 모듈
- 조직 간 벤치마킹, 다년 추세, 시나리오 비교
- 정기 보고서 자동 생성 및 알림
- 대학 내 기존 시스템/회계 시스템 연계
- AI 기반 경영 브리핑 및 이상 징후 설명

### 4.3 제외 범위

- 1차 구축에서 대학 회계·인사 원천 시스템을 대체하지 않는다.
- 개인정보·민감 연구자료 원문을 무분별하게 복제하지 않는다.
- 공개 홈페이지를 만들지 않는다. 기본값은 교내 승인 사용자 전용이다.

## 5. 정보 구조 기준

1. 조직은 현재 조직도를 기준으로 계층형 마스터로 관리하되 개편 이력을 보존한다.
2. 사업은 특정 조직에 귀속되지만 여러 조직이 참여할 수 있다.
3. 지표는 전사 공통지표와 조직/사업별 지표를 함께 지원한다.
4. 문서는 폴더를 흉내 내는 데 그치지 않고 조직, 업무영역, 사업, 회의, 지표에 복수 연결한다.
5. 모든 수치는 기준일, 출처, 입력자, 검증상태를 가진다.

## 8. 성공 기준

- 단장이 3회 이내의 클릭으로 모든 조직/사업의 핵심 현황과 근거에 도달한다.
- 월간 보고용 핵심 지표의 90% 이상이 대시보드에서 조회된다.
- 예산·KPI 카드의 100%가 기준일과 원천/근거를 표시한다.
- 일반 사용자는 허용된 조직 범위 밖의 행을 직접 API 호출로도 조회할 수 없다.
- 주요 수치의 변경 이력이 감사 로그로 재현된다.
- 10명의 대표 사용자 시나리오에서 핵심 과업 성공률 90% 이상을 달성한다.

## 9. 위험과 대응

| 위험 | 영향 | 대응 |
|---|---|---|
| 조직·사업별 지표 정의가 서로 다름 | 비교 왜곡 | 공통 지표 사전과 단위/산식/주기 승인 절차 도입 |
| 기존 엑셀/문서 데이터 품질 편차 | 신뢰도 하락 | 단계별 반입, 검증상태, 오류 리포트, 데이터 오너 지정 |
| 개인정보·협약정보 노출 | 법적/평판 위험 | 분류등급, RLS, 다운로드 권한, 감사 로그, 보존정책 적용 |
| 화면에 정보가 과밀해짐 | 사용성 저하 | 단장 요약→조직/사업→근거의 3단계 점진 공개 |
| 조직 개편으로 과거 비교 단절 | 추세 왜곡 | 조직 유효기간과 승계 관계 보존 |

## 11. 구현 전 확정이 필요한 정책

- 사용자 계정 체계(교직원 SSO 연계 여부, 이메일 도메인)
- 실제 조직별 데이터 오너와 승인자
- 회계연도/사업연도 정의 및 금액 단위
- 핵심 경영지표 10~20개와 목표 산식
- 문서 저장 위치(Supabase Storage 또는 기존 Drive/그룹웨어 링크)
- 개인정보/대외비 분류, 보존기간, 다운로드 제한
